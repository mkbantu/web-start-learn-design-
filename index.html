What is the Internet?

The Internet is a global network of interconnected computers that communicate using standard protocols. It is the largest network ever built — connecting billions of public, private, academic, business, and government networks around the world.

Definition: The Internet is a GSI — Global System of Interconnected networks. It is a worldwide system of computer networks — a network of networks — where users at any computer can access information and communicate with others at any other computer, regardless of location.

A Brief History

1966 — ARPANET

The Internet began as ARPANET (Advanced Research Projects Agency Network). The concept was proposed and planned in 1966–1967 by the US Department of Defense's Advanced Research Projects Agency. By 1969, the first wide area network (WAN) became operational, initially connecting four university sites: UCLA, Stanford Research Institute, UC Santa Barbara, and the University of Utah. It was the world's first packet-switching network.

1989 — World Wide Web

Timothy Berners-Lee invented the World Wide Web at CERN in Switzerland. He created HTML, HTTP, and the first web browser — making it possible to share and navigate information on the Internet through clickable links.

Today — Global Network

Over 5 billion people use the Internet today. It powers communication, business, education, entertainment, and government — all accessed through web browsers, apps, and connected devices.

Uses of the Internet

The Internet has transformed how we live, learn, work, and connect with others. Its main uses fall into four broad categories:

The four main Internet tools:
  • Information Retrieval — searching for and accessing information using search engines (Google, Bing), online encyclopedias, databases, and digital libraries
  • Communication — sending email, instant messaging, video calls, and participating in social media platforms
  • Multimedia — streaming video and audio, publishing photos, watching live broadcasts, and accessing online games
  • Search — finding specific content, products, people, and services through keyword searches and directories

Communication

Send emails, instant messages, and video calls (Zoom, WhatsApp, Google Meet). Social media platforms like Twitter, Facebook, and LinkedIn connect billions of people globally.

Email: An email address has the format user_name@domain_name (e.g. alice@gmail.com). Messages travel between mail servers using the SMTP protocol and are collected by the recipient using POP3 or IMAP.

Research & Education

Access libraries, academic journals, online courses, and encyclopedias from anywhere. Platforms like Google Scholar, Coursera, Khan Academy, and YouTube deliver free education worldwide.

E-Commerce

Buy and sell goods and services online. Amazon, Jumia, eBay, and millions of independent shops operate entirely over the Internet, reaching customers anywhere on Earth.

Entertainment

Stream movies and series (Netflix), listen to music (Spotify), watch videos (YouTube), play online games, and read news — all delivered over the Internet in real time.

Cloud Storage

Store files online (Google Drive, OneDrive, Dropbox) and access them from any device. Collaborate on documents, spreadsheets, and presentations with people anywhere.

Remote Work

Work from home, collaborate with global teams, attend virtual meetings, and manage entire businesses — all made possible by the Internet and web-based tools.

Properties of the Internet

Four key properties define how the Internet works and why it is so powerful:

Interoperable

Computers running Windows, macOS, Linux, iOS, or Android can all communicate with each other using shared, open standards and protocols — regardless of manufacturer or operating system.

Global

There are no geographic boundaries. A website hosted in Rwanda can be accessed instantly from Japan, Brazil, or anywhere else. The network spans every country and continent.

Cost-Shared

No single company or government owns the Internet. The cost of maintaining it is shared across millions of ISPs, organisations, and governments, making it affordable and resilient.

Heterogeneous

The Internet is built from many different types of hardware, software, and network technologies — from fibre-optic cables to satellite links — all working together through common protocols.

How the Web Works

The World Wide Web (WWW) is a system of web pages stored on servers and accessed through the Internet using a browser. When you visit a website, your browser sends a request to a server, and the server sends back the HTML, CSS, and JavaScript files that make up the page.

The Web is not the same as the Internet. The Internet is the physical infrastructure — cables, routers, and servers. The Web is a service that runs on top of the Internet, alongside email, file transfer, and other services.

IP Addresses

Every device on the Internet has a unique IP address (Internet Protocol address) — like a postal address for computers.

IPv4 example: 192.168.1.1
IPv6 example: 2001:db8::1

IPv4 uses four numbers (0–255) separated by dots. IPv6 is newer and much longer, supporting trillions more devices as the Internet grows.

Domain Names & DNS

Because IP addresses are hard to remember, we use domain names — human-readable names like google.com or ur.ac.rw.

A DNS (Domain Name System) server works like a phone book, automatically translating domain names into IP addresses every time you type a URL.

Example: www.google.com → 142.250.80.36

URLs — Web Addresses

A URL (Uniform Resource Locator) is the full address of any resource on the Web. Every URL has this structure:

https://www.example.com/page/name
      ↑ Protocol          ↑ Domain name              ↑ File path

Top-Level Domains (TLDs)

The last part of a domain name is the Top-Level Domain (TLD). Different TLDs indicate the type or country of the website:

.com

Commercial websites and businesses worldwide. The most common TLD.

.org

Non-profit organisations, charities, and open communities.

.edu

Educational institutions, mainly universities in the USA.

.gov

Government websites — federal, state, and local government agencies.

.net

Originally for network providers. Now used widely for various purposes.

.rw

Country code for Rwanda. Used by Rwandan individuals, businesses, and institutions.

Internet Protocols

A protocol is a set of agreed rules that defines how data is sent and received over the Internet. Different types of communication use different protocols:

Protocol Full Name What It Does
HTTP HyperText Transfer Protocol Transfers web pages from a server to your browser. The foundation of the World Wide Web. Every web address starts with http://.
HTTPS HTTP Secure Same as HTTP but fully encrypted. Required for banking, shopping, and login pages. Indicated by a padlock icon in the browser and https:// in the URL.
SMTP Simple Mail Transfer Protocol Sends email messages between mail servers. Used every time you send an email, whether on Gmail, Outlook, or any mail client.
FTP File Transfer Protocol Transfers files between computers over a network. Web designers use FTP to upload their HTML, CSS, and image files to a web server when publishing a site.
TCP/IP Transmission Control Protocol / Internet Protocol The core language of the Internet. TCP breaks data into packets and reassembles them. IP routes each packet across the network to its destination address.
Remember: Every time you load a webpage, your browser and the server are communicating using HTTP or HTTPS. You can see the protocol at the very start of every URL in your browser's address bar.

Static vs Dynamic Websites

There are two types of websites. Understanding the difference will help you know what to learn and when:

Static Websites

Pages that show the same content to every visitor. The content only changes when the developer manually edits and re-uploads the HTML file.

  • Built with HTML and CSS only
  • No server-side processing needed
  • Fast to load, easy to host
  • No database required
  • Good for portfolios, landing pages, course sites
  • Examples: personal portfolio, this course site

Dynamic Websites

Pages that generate content on demand — customised per user, time, or database query. The server builds the page fresh each time someone visits.

  • HTML + CSS + JavaScript + server-side language
  • Uses a database (MySQL, MongoDB)
  • Can personalise content per logged-in user
  • Handles user accounts, purchases, comments
  • Good for social media, e-commerce, news sites
  • Examples: Facebook, Amazon, Gmail, YouTube
Where to start: Begin with static websites using HTML and CSS. Once you can build and style pages confidently, add JavaScript for interactivity — then learn back-end languages to build fully dynamic sites.

The Evolution of the Web

Since Timothy Berners-Lee invented the Web at CERN in 1989, it has gone through three major phases:

1.0

Web 1.0

1991 – 2004  ·  Read-Only Web

Static pages of text and images — sometimes called "brochure-ware" because sites simply advertised information without any interaction. Visitors could only read; there were no user accounts, no comments, and no communities. Like an online library or newspaper that never changes.

2.0

Web 2.0

2004 – present  ·  Read-Write Web

Interactive websites where users create and share content. Web 2.0 introduced social media (Facebook, Twitter, YouTube), user-generated content, tagging (labelling content with keywords), keyword search (Google indexing user content), blogs, and wikis. Powered by JavaScript and server-side languages.

3.0

Web 3.0

Emerging  ·  Semantic & Decentralised

AI-powered, personalised, and decentralised. Machines understand the meaning of content. Blockchain applications, smart contracts, and no central authority.

Web Technologies

Building a website uses two layers of technology. You must understand both — even if you only build one side:

Front-End  —  Client Side

These technologies run in the browser on the visitor's device. Every website in the world uses all three:

HTML CSS JavaScript

  • HTML — The structure and content of every page (headings, paragraphs, images, links, forms)
  • CSS — The visual design: colours, fonts, layout, spacing, animations, and responsive design
  • JavaScript — Behaviour and interactivity: buttons, form validation, live content updates, and animations

Back-End  —  Server Side

These technologies run on the server, invisible to visitors. They power dynamic content and databases:

PHP Python MySQL

  • PHP — Server-side scripting; generates HTML pages dynamically from database data
  • Python — Web frameworks such as Django and Flask; popular for data-driven and AI-powered sites
  • MySQL — Relational database; stores user accounts, posts, products, orders, and all application data
This course covers front-end development — HTML, CSS, and JavaScript. These are the essential foundations every web developer must master before learning back-end technologies. Master them first.

Tools You Will Use

Every web developer uses these tools. Most are completely free to download and use:

Text Editor

Where you write code. Use Visual Studio Code (VS Code) — it is free, powerful, has built-in Git support, and is the most popular code editor in the world among web developers.

Web Browser

Where you test your pages. Google Chrome or Mozilla Firefox both have built-in developer tools. Press F12 to open DevTools and inspect your HTML and CSS live.

Web Server

A computer that stores and delivers your web files. While learning, your own computer acts as the server. To publish online, use a free host like Netlify or GitHub Pages.

Git & GitHub

Git tracks changes in your code so you can undo mistakes. GitHub stores your projects online, lets you collaborate with others, and is where employers look at your work.

CodePen

An online playground for HTML, CSS, and JavaScript — no installation needed. Write code in the browser and see the result instantly. Perfect for experimenting and sharing small projects.

Browser DevTools

Built into Chrome and Firefox (press F12). Lets you inspect HTML elements, edit CSS live, test mobile layouts, debug JavaScript, and monitor network requests — all in real time.
