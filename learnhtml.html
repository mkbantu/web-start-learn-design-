HTML Introduction

HTML (HyperText Markup Language) is the standard language for creating web pages. It describes page structure using elements represented by tags.

<!DOCTYPE html>
<html>
<head><title>Page Title</title></head>
<body>
  <h1>My First Heading</h1>
  <p>My first paragraph.</p>
</body>
</html>
  • HTML stands for HyperText Markup Language
  • HTML describes the structure of web pages using elements
  • Browsers do not display tags — they use them to render content

HTML Editors

HTML can be written in any text editor. Popular choices include VS Code, Sublime Text, and Notepad++.

  • Recommended: VS Code — free, powerful, with extensions
  • Save files with a .html extension
  • Open .html files in a browser to preview them
  • Do NOT use word processors like Word — they add hidden formatting

HTML Basic

Every HTML page follows the same basic skeleton structure.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
</head>
<body>
  <h1>Heading</h1>
  <p>Paragraph.</p>
</body>
</html>
  • <!DOCTYPE html> — declares HTML5 document
  • <html> — root element of the page
  • <head> — contains metadata (not visible)
  • <body> — contains everything visible on screen

HTML Elements

An HTML element has a start tag, content, and an end tag. Some elements are self-closing (void elements).

<tagname>Content goes here</tagname>

<h1>My Heading</h1>
<p>My paragraph.</p>
<br>   <!-- self-closing -->
<img src="photo.jpg" alt="Photo">
  • Elements can be nested inside each other
  • Never skip end tags (except void elements like <br>, <img>)
  • HTML is not case-sensitive, but lowercase is recommended

HTML Attributes

Attributes provide extra information about elements. They appear in the opening tag as name/value pairs.

<a href="https://example.com">Visit</a>
<img src="photo.jpg" alt="A photo" width="300">
<p style="color:red;">Red text</p>
<input type="text" placeholder="Enter name">
  • Always use lowercase attribute names
  • Always quote attribute values
  • Common: href, src, alt, style, id, class

HTML Headings

HTML has six heading levels, <h1> to <h6>. <h1> is most important, <h6> least.

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
  • Use headings for structure, not just to make text big
  • Each page should have only one <h1>
  • Headings are block-level elements with default top/bottom margins

HTML Paragraphs

The <p> element defines a paragraph. Browsers automatically add space before and after it.

<p>This is a paragraph.</p>
<p>Another paragraph.</p>
<hr>                        <!-- horizontal rule -->
<p>Line one.<br>Line two.</p>  <!-- line break -->
<pre>  Preserved
  whitespace</pre>
  • Extra spaces and newlines in HTML collapse to one space
  • <br> — line break without a new paragraph
  • <hr> — thematic break / horizontal line
  • <pre> — preserves spaces and line breaks exactly

HTML Styles

The style attribute adds inline CSS directly to an element.

<p style="color:blue; font-size:20px;">Blue text</p>
<h1 style="background-color:yellow;">Yellow bg</h1>
<p style="text-align:center;">Centered</p>
<p style="font-family:courier;">Courier font</p>
  • Syntax: style="property: value;"
  • Use sparingly — prefer external CSS files for maintainability
  • Common properties: color, background-color, font-size, text-align

HTML Formatting

Formatting elements give text special visual meaning. Prefer semantic tags over purely visual ones.

<b>Bold</b>            <strong>Important</strong>
<i>Italic</i>           <em>Emphasized</em>
<mark>Highlighted</mark>
<small>Small text</small>
<del>Deleted text</del> <ins>Inserted text</ins>
<sub>Subscript</sub>    <sup>Superscript</sup>
  • <b> = bold (visual); <strong> = important (semantic)
  • <i> = italic (visual); <em> = emphasis (semantic)
  • Prefer <strong> and <em> — screen readers announce them differently

HTML Quotations

HTML provides specific elements for citing and quoting content.

<blockquote cite="https://source.com">
  Long quoted passage here.
</blockquote>

<p>Einstein said <q>E=mc²</q>.</p>

<p><abbr title="HyperText Markup Language">HTML</abbr></p>

<address>Written by John Doe.</address>

<p><cite>The Scream</cite> by Edvard Munch.</p>
  • <blockquote> — block-level; browsers indent it
  • <q> — inline; browsers add quotation marks
  • <abbr> — abbreviation; title shows full form on hover
  • <cite> — title of a creative work (italic by default)

HTML Comments

Comments are not displayed by the browser. They document code or temporarily hide content.

<!-- This is a comment -->
<p>Visible paragraph.</p>
<!-- <p>Hidden paragraph.</p> -->

<!--
  Multi-line comment
  spanning several lines
-->
  • Syntax: <!-- ... -->
  • Comments are still visible in page source — never put secrets in them
  • Useful for debugging: temporarily hide elements without deleting them

HTML Colors

Colors can be specified by name, RGB, HEX, or HSL — on text, backgrounds, and borders.

<p style="color:Tomato;">Named color</p>
<p style="color:rgb(255,99,71);">RGB</p>
<p style="color:#ff6347;">HEX</p>
<p style="color:hsl(9,100%,64%);">HSL</p>
<p style="background-color:lightblue;">Background</p>
  • HTML supports 140+ named colors (e.g. Red, Coral, Navy)
  • Applied to: color (text), background-color, border-color

Colors — RGB & RGBA

RGB mixes Red, Green, Blue values (0–255). RGBA adds an alpha (opacity) channel 0.0–1.0.

color: rgb(255, 0, 0);       /* red */
color: rgb(0, 0, 255);       /* blue */
color: rgb(0, 0, 0);         /* black */
color: rgb(255, 255, 255);   /* white */
color: rgba(255, 0, 0, 0.5); /* 50% transparent red */

Colors — HEX

HEX colors use #RRGGBB — two hex digits per channel (00–ff).

#ff0000  /* red */
#00ff00  /* green */
#0000ff  /* blue */
#000000  /* black */
#ffffff  /* white */
#f00     /* shorthand = #ff0000 */
  • Shorthand works when both digits of each pair match: #abc = #aabbcc

Colors — HSL & HSLA

HSL = Hue (0–360°), Saturation (0–100%), Lightness (0–100%). HSLA adds alpha.

color: hsl(0, 100%, 50%);      /* red */
color: hsl(120, 100%, 50%);    /* green */
color: hsl(240, 100%, 50%);    /* blue */
color: hsla(0, 100%, 50%, 0.3);/* 30% transparent */
  • Hue: 0/360 = red, 120 = green, 240 = blue
  • Saturation 0% = grey, 100% = full color
  • Lightness 0% = black, 50% = normal, 100% = white

HTML CSS

CSS styles HTML. It can be inline, internal (in a <style> tag), or external (linked file).

<!-- Inline -->
<p style="color:red;">Red</p>

<!-- Internal (in <head>) -->
<style>
  p { color: red; }
</style>

<!-- External (recommended) -->
<link rel="stylesheet" href="styles.css">
  • External CSS is the best practice — one file, multiple pages
  • Inline overrides internal, which overrides external
  • CSS rule: selector { property: value; }

HTML Links

The <a> tag creates hyperlinks. The href attribute specifies the destination.

<a href="https://example.com">Visit Example</a>
<a href="https://example.com" target="_blank">New tab</a>
<a href="page2.html">Local page</a>
<a href="mailto:me@email.com">Email me</a>
<a href="tel:+1234567890">Call me</a>
  • target="_blank" — opens in a new tab
  • Absolute URL: full address with https://
  • Relative URL: path relative to current page

Link Colors

Links have four CSS states that can each be styled separately.

a:link    { color: blue; }    /* unvisited */
a:visited { color: purple; }  /* visited */
a:hover   { color: red; }     /* on hover */
a:active  { color: orange; }  /* being clicked */
  • Order matters — use LoVe HAte: link, visited, hover, active
  • Default: unvisited = blue underline, visited = purple

Link Bookmarks

Bookmark links jump to a specific element on the page using its id.

<!-- Target element -->
<h2 id="section2">Section 2</h2>

<!-- Same-page link -->
<a href="#section2">Go to Section 2</a>

<!-- From another page -->
<a href="page.html#section2">Go to Section 2</a>

HTML Images

The <img> tag embeds an image. It is a void (self-closing) element.

<img src="photo.jpg" alt="Description" width="500" height="300">
  • src — path to the image (required)
  • alt — text shown if image fails; required for accessibility
  • Always set width and height to prevent layout shift on load
  • Formats: JPEG, PNG, GIF, SVG, WebP

Image Map

Image maps define clickable areas on an image using <map> and <area>.

<img src="world.jpg" usemap="#worldmap" alt="World">
<map name="worldmap">
  <area shape="rect"   coords="34,44,270,350" href="europe.html" alt="Europe">
  <area shape="circle" coords="90,58,30"       href="africa.html" alt="Africa">
  <area shape="poly"   coords="140,121,181,116,204,160" href="asia.html" alt="Asia">
</map>
  • Shapes: rect (x1,y1,x2,y2), circle (x,y,r), poly (x1,y1,x2,y2,...)

Background Images

Background images are applied with CSS, not <img>.

/* CSS */
body {
  background-image: url('bg.jpg');
  background-repeat: no-repeat;
  background-size: cover;
  background-position: center;
}
  • no-repeat — show once; repeat-x / repeat-y — tile one axis
  • cover — scale to fill, may crop; contain — fit inside, may leave gaps

The Picture Element

<picture> serves different images based on screen size or format support.

<picture>
  <source media="(min-width:768px)" srcset="large.jpg">
  <source media="(min-width:480px)" srcset="medium.jpg">
  <img src="small.jpg" alt="A flower">
</picture>
  • Browser picks the first matching <source>
  • <img> is the fallback — always required
  • Also used to offer WebP with a JPEG fallback

HTML Favicon

A favicon is the small icon shown in the browser tab.

<head>
  <link rel="icon" type="image/x-icon" href="/favicon.ico">
</head>
  • Recommended size: 16×16 or 32×32 pixels
  • Formats: .ico, .png, .svg
  • Place in your root folder so browsers find it automatically

HTML Page Title

The <title> in <head> sets the tab title and is important for SEO.

<head>
  <title>My Website — Home</title>
</head>
  • Required in every HTML document
  • Search engines display it as the clickable headline in results
  • Keep it descriptive and under ~60 characters

HTML Tables

Tables organize data into rows and columns.

<table>
  <tr>
    <th>Name</th><th>Age</th>
  </tr>
  <tr>
    <td>Alice</td><td>25</td>
  </tr>
</table>
  • <tr> = row, <th> = header cell (bold + centered), <td> = data cell
  • Use tables for tabular data only, not for page layout

Table Borders

table, th, td {
  border: 1px solid black;
  border-collapse: collapse; /* removes double borders */
}
th, td { border-radius: 4px; }
  • border-collapse: collapse — merges adjacent borders into one
  • border-collapse: separate — keeps individual borders (default)

Table Sizes

table        { width: 100%; }
th, td       { height: 50px; }
th:first-child { width: 60%; }
th:last-child  { width: 40%; }

Table Headers

Use <caption> to add a title, and <thead>/<tbody>/<tfoot> for structure.

<table>
  <caption>Monthly Savings</caption>
  <thead>
    <tr><th>Month</th><th>Savings</th></tr>
  </thead>
  <tbody>
    <tr><td>January</td><td>$100</td></tr>
  </tbody>
  <tfoot>
    <tr><td>Total</td><td>$100</td></tr>
  </tfoot>
</table>

Padding & Spacing

th, td { padding: 15px; }           /* space inside cell */
table  { border-spacing: 10px; }    /* space between cells */
  • border-spacing only works when border-collapse: separate

Colspan & Rowspan

<td colspan="2">Spans 2 columns</td>
<td rowspan="3">Spans 3 rows</td>

Table Styling

tr:nth-child(even) { background-color: #f2f2f2; } /* zebra stripes */
tr:hover           { background-color: #ddd; }      /* hover highlight */
th, td             { border-bottom: 1px solid #ddd; }/* horizontal lines only */

Table Colgroup

<colgroup> applies styles to entire columns without repeating them on each cell.

<table>
  <colgroup>
    <col span="2" style="background-color:yellow">
    <col style="background-color:pink">
  </colgroup>
  ...
</table>
  • Only width, visibility, background, and border work on <col>

HTML Lists

HTML has three list types: unordered (<ul>), ordered (<ol>), and description (<dl>).

<ul><li>Coffee</li><li>Tea</li></ul>
<ol><li>First</li><li>Second</li></ol>

Unordered Lists

<ul> creates a bulleted list. Bullet style is controlled by CSS.

ul { list-style-type: disc; }    /* default ● */
ul { list-style-type: circle; }  /* ○ */
ul { list-style-type: square; }  /* ■ */
ul { list-style-type: none; }    /* no bullet */

Ordered Lists

<ol> creates a numbered list. Type and start value can be customized.

<ol type="1"> <!-- 1, 2, 3 (default) -->
<ol type="A"> <!-- A, B, C -->
<ol type="a"> <!-- a, b, c -->
<ol type="I"> <!-- I, II, III -->
<ol start="5"><!-- starts at 5 -->

Description Lists

<dl> pairs terms (<dt>) with their descriptions (<dd>).

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

Block & Inline Elements

Block elements start on a new line and take full width. Inline elements stay on the same line.

<!-- Block: div, p, h1-h6, ul, ol, table, form, section -->
<!-- Inline: span, a, img, strong, em, label, input -->
  • Block elements can contain both inline and block elements
  • Inline elements should only contain inline elements or text
  • CSS display property can change an element's type

HTML Div

<div> is a generic block-level container used to group elements for styling.

<div style="background:lightblue; padding:20px;">
  <h2>Title</h2>
  <p>Paragraph inside a div.</p>
</div>
  • No visual style of its own — it is a grouping/layout tool
  • Use with class or id to apply CSS

HTML Classes

The class attribute targets multiple elements with the same CSS style.

<p class="highlight">First</p>
<p class="highlight note">Two classes</p>

.highlight { background-color: yellow; }
.note      { font-style: italic; }
  • Multiple elements can share a class
  • One element can have multiple classes (space-separated)
  • Class names are case-sensitive

HTML Id

The id attribute uniquely identifies a single element on the page.

<h1 id="main-title">Hello</h1>

#main-title { color: navy; }

document.getElementById("main-title").style.color = "red";
  • Each id must be unique — only one per page
  • Used for CSS targeting, JS selection, and bookmark links (href="#id")

HTML Buttons

<button>Click Me</button>
<button type="submit">Submit</button>
<button type="reset">Reset</button>
<button type="button" onclick="alert('Hi')">Alert</button>
<button disabled>Disabled</button>
  • type="submit" — submits the form (default inside a form)
  • type="reset" — clears all form fields
  • type="button" — no default action; use with JavaScript

HTML Iframes

<iframe> embeds another HTML page inside the current one.

<iframe src="https://example.com" width="600" height="400" title="Example"></iframe>
<iframe src="page.html" style="border:none;"></iframe>
  • Always include a title attribute for accessibility
  • Many websites block embedding via X-Frame-Options headers
  • Can be a link target: <a href="page.html" target="myFrame">

HTML JavaScript

The <script> tag embeds or links JavaScript.

<script>
  document.getElementById("demo").innerHTML = "Hello!";
</script>

<script src="app.js" defer></script>

<noscript>Enable JavaScript to use this site.</noscript>
  • Place <script> before </body>, or use the defer attribute for faster load
  • defer — runs after HTML is parsed; keeps execution order
  • async — runs as soon as downloaded; order not guaranteed

HTML File Paths

<img src="https://example.com/photo.jpg">  <!-- absolute -->
<img src="photo.jpg">                       <!-- same folder -->
<img src="images/photo.jpg">               <!-- subfolder -->
<img src="../photo.jpg">                    <!-- parent folder -->
<img src="/images/photo.jpg">              <!-- from site root -->
  • Prefer relative paths — they work on any domain
  • ../ moves up one folder level

HTML Head

<head> holds metadata — not displayed on the page.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Free web tutorials">
  <title>My Page</title>
  <link rel="stylesheet" href="styles.css">
  <script src="app.js" defer></script>
</head>
  • charset="UTF-8" — must be first meta tag
  • viewport meta is essential for responsive design
  • <base href="/"> — sets default URL for all relative links

HTML Layout

HTML5 semantic elements define page regions. Use CSS Flexbox or Grid to position them.

<header>  — top of page
<nav>     — navigation links
<main>    — main content (one per page)
<section> — thematic grouping
<article> — self-contained content
<aside>   — sidebar
<footer>  — bottom of page
  • Semantic tags improve SEO and accessibility
  • Flexbox: display: flex — best for one-dimensional layout
  • Grid: display: grid — best for two-dimensional layout

HTML Responsive

Responsive design makes pages look good on all screen sizes.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

/* Flexible image */
img { max-width: 100%; height: auto; }

/* Media query */
@media (max-width: 600px) {
  body { font-size: 14px; }
  .col { width: 100%; }
}
  • Always include the viewport meta tag
  • Use max-width: 100% on images to prevent overflow
  • Use relative units: %, em, rem, vw, vh

HTML Computercode

<code>x = 5;</code>                   <!-- inline code -->
<pre><code>x = 5;
y = 10;</code></pre>                  <!-- code block -->
<kbd>Ctrl + C</kbd>                    <!-- keyboard input -->
<samp>Error: file not found</samp>     <!-- sample output -->
<var>x</var> = <var>y</var> + 2       <!-- variable -->

HTML Semantics

Semantic elements describe their meaning to the browser and developer.

<!-- Non-semantic -->
<div id="nav">...</div>

<!-- Semantic -->
<nav>...</nav>
<article>...</article>
<figure>
  <img src="cat.jpg" alt="Cat">
  <figcaption>A cute cat.</figcaption>
</figure>
<time datetime="2026-05-07">May 7, 2026</time>
  • Better SEO — search engines understand page structure
  • Better accessibility — screen readers navigate by landmarks

HTML Style Guide

  • Always declare <!DOCTYPE html>
  • Use lowercase element and attribute names
  • Always close all HTML elements
  • Always quote attribute values
  • Always specify alt, width, height on images
  • Avoid inline styles — use CSS files
  • Indent nested elements with 2 spaces
  • Add lang attribute to <html>
  • Use <meta charset="UTF-8"> as first item in <head>

HTML Entities

Entities display reserved characters that would otherwise be parsed as HTML.

&lt;    → <      &gt;   → >
&amp;   → &      &quot; → "
&nbsp;  → (non-breaking space)
&copy;  → ©      &reg;  → ®
&trade; → ™      &euro; → €
  • Format: &name; or &#number;
  • &nbsp; prevents a line break between two words

HTML Symbols

&#8704; → ∀   &#8706; → ∂   &#8707; → ∃
&#8364; → €   &#9742; → ☎   &#9824; → ♠
&#9829; → ♥   &#9827; → ♣   &#9830; → ♦

HTML Emojis

Emojis are Unicode characters. Ensure charset="UTF-8" is set.

<meta charset="UTF-8">
<p>😀</p>           <!-- direct -->
<p>&#128512;</p>     <!-- decimal entity -->
<p>&#x1F600;</p>    <!-- hex entity -->

HTML Charsets

A charset defines how characters are encoded. Always use UTF-8.

<meta charset="UTF-8">
  • UTF-8 is the HTML5 default and covers all languages + emojis
  • ASCII supports only 128 characters (English only)
  • Omitting charset can cause special characters to display as garbage

HTML URL Encode

URLs use ASCII only. Non-ASCII characters are percent-encoded.

Space → %20    ! → %21    # → %23
%     → %25    & → %26    = → %3D
?     → %3F    @ → %40
  • Space can also be encoded as + in query strings
  • Browsers encode URLs automatically before sending

HTML vs. XHTML

XHTML is HTML written as stricter XML. HTML5 is now the standard.

HTML5XHTML
Tags may be uppercaseAll tags must be lowercase
End tags optional for someAll tags must be closed
Attribute quotes optionalAll attributes must be quoted
<br><br />

Best practice: follow XHTML-style rules even in HTML5.

HTML Forms

The <form> element collects user input and sends it to a server.

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">
  <button type="submit">Submit</button>
</form>
  • method="get" — data visible in URL (bookmarkable, not for sensitive data)
  • method="post" — data in request body (secure, for passwords/files)

HTML Form Attributes

<form
  action="/submit"
  method="post"
  autocomplete="on"
  novalidate
  target="_blank"
  enctype="multipart/form-data"
>
  • novalidate — skip browser built-in validation
  • enctype="multipart/form-data" — required for file uploads

HTML Form Elements

<input>      various input types
<label>      label for an input
<select>     dropdown list
<textarea>   multi-line text
<button>     clickable button
<fieldset>   groups related inputs
<legend>     caption for fieldset
<datalist>   predefined suggestions for input
<output>     result of a calculation
<textarea rows="4" cols="30">Default text</textarea>

<select name="car">
  <optgroup label="Swedish">
    <option value="volvo">Volvo</option>
  </optgroup>
</select>

<input type="text">       single-line text
<input type="password">   masked input
<input type="email">      validates email
<input type="number">     numeric

<input type="checkbox">   tick box
<input type="radio">      one-of-many choice

<input type="color">      color picker

Input Form Attributes

Inputs outside a <form> tag can still be linked using the form attribute.

<form id="myForm" action="/submit">
  <button type="submit">Submit</button>
</form>
<input type="text" name="extra" form="myForm">
  • formaction — override the form's action for one button
  • formmethod — override the form's method
  • formnovalidate — skip validation for this button only

HTML Canvas

<canvas> draws pixel-based graphics via JavaScript.

<canvas id="c" width="400" height="200"></canvas>
<script>
  const ctx = document.getElementById("c").getContext("2d");
  ctx.fillStyle = "red";
  ctx.fillRect(10, 10, 150, 80);   /* rectangle */
  ctx.beginPath();
  ctx.arc(200, 100, 50, 0, 2*Math.PI);
  ctx.stroke();                    /* circle */
</script>

HTML SVG

SVG draws vector graphics in XML. Scales perfectly at any size.

<svg width="200" height="200">
  <circle cx="100" cy="100" r="80" fill="green"/>
  <rect x="10" y="10" width="80" height="50" fill="blue"/>
  <line x1="0" y1="0" x2="200" y2="200" stroke="red"/>
  <text x="60" y="110" fill="white">SVG</text>
</svg>
  • SVG = best for logos, icons, charts (scales without blur)
  • Canvas = best for games, image manipulation (pixel-based)
  • SVG elements are part of the DOM — styleable and scriptable

HTML Web APIs

Web APIs are browser-built-in interfaces accessible via JavaScript.

  • Geolocation — get user's GPS position
  • Web Storage — store data locally (localStorage / sessionStorage)
  • Drag and Drop — drag elements on the page
  • Web Workers — run JS in a background thread
  • SSE — server pushes real-time data to browser

HTML Geolocation

if (navigator.geolocation) {
  navigator.geolocation.getCurrentPosition(pos => {
    console.log(pos.coords.latitude, pos.coords.longitude);
  });
}
  • Browser always requests user permission first
  • Works best on mobile devices with GPS

HTML Drag and Drop

<div draggable="true" ondragstart="drag(event)" id="item">Drag me</div>
<div ondrop="drop(event)" ondragover="allowDrop(event)">Drop here</div>

function allowDrop(e) { e.preventDefault(); }
function drag(e)      { e.dataTransfer.setData("text", e.target.id); }
function drop(e) {
  e.preventDefault();
  const id = e.dataTransfer.getData("text");
  e.target.appendChild(document.getElementById(id));
}

HTML Web Storage

Stores key/value pairs in the browser. Larger and more secure than cookies.

/* localStorage — persists after tab/browser close */
localStorage.setItem("name", "Alice");
localStorage.getItem("name");    /* "Alice" */
localStorage.removeItem("name");

/* sessionStorage — cleared when tab closes */
sessionStorage.setItem("temp", "value");
  • ~5–10 MB limit per origin
  • Values are strings — use JSON.stringify() for objects

HTML Web Workers

Web Workers run JavaScript in a background thread so the UI stays responsive.

/* main.js */
const worker = new Worker("worker.js");
worker.postMessage("start");
worker.onmessage = e => console.log("Result:", e.data);

/* worker.js */
onmessage = e => postMessage(heavyTask());
  • Workers cannot access the DOM
  • Communicate via postMessage() / onmessage

HTML SSE (Server-Sent Events)

SSE lets a server push data to the browser over a persistent HTTP connection.

/* Browser */
const source = new EventSource("/events");
source.onmessage = e => console.log("Data:", e.data);

/* Server (Node.js) */
res.setHeader("Content-Type", "text/event-stream");
res.write("data: Hello\n\n");
  • One-directional: server → browser only
  • Auto-reconnects if connection drops
  • For two-way communication use WebSockets instead
